## Структура кода игры:

### Core - Ядро и управление

- **`main.py`**: Точка входа. Создаёт объект `Game` и запускает его. Больше ничего не делает.
- **`config.py`**: Хранилище настроек (FPS, размеры экрана, цвета, баланс: урон, скорость). *Никакого кода, только переменные.*
- **`game.py`**: Главный менеджер. Управляет временем (DT), переключает состояния (меню/игра) и связывает `World`, `Renderer` и `Camera` воедино.

### Models

- **`world.py`**: Контейнер. Хранит списки всех врагов, пуль, игрока и бур. Обсчитывает их столкновения (физику) и удаляет мёртвых.
- **`entity.py`**: Базовый чертёж. Хранит общие поля (HP, позиция, скорость) и базовый метод движения `move()`.
- **`drone.py`**: Логика дрона. Управляет инвентарём (медь) и стрельбой.
- **`drill.py`**: Логика бура. Считает прочность, движется вглубь, "съедает" тайлы сетки перед собой.
- **`enemy.py`**: Искусственный интеллект. Решает, куда бежать (к буру или игроку) и когда атаковать.
- **`map.py`**: Карта мира. Хранит массив тайлов (камень, медь, пустота). Знает, что находится в конкретной точке координат.

### Views

- **`renderer.py`**: Художник. Получает объекты из `World`, спрашивает у `Camera` их экранные координаты и рисует картинки (спрайты) на экран.
- **`camera.py`**: Математик. Вычисляет "сдвиг" мира. Следит за игроком, чтобы он всегда был в центре экрана, и выдаёт остальным модулям правильные координаты для отрисовки.

### Utils - Инструменты

- **`math_helpers.py`**: Вспомогательные функции (например, расчёт дистанции между врагом и игроком).
- **`pathfinding.py`**: Твой сложный алгоритм (A\*). Принимает сетку `Grid`, точку А и точку Б — возвращает путь для врага.
